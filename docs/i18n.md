## Quick Start

The essential steps for adding i18n to any Compose project:

1. **Add default strings** to your resources folder
2. **Create translations** in locale-specific folders
3. **Use `stringResource()`** to display strings in your composables

```kotlin
// Basic usage
Text(text = stringResource(R.string.hello_world))

// With parameters
Text(text = stringResource(R.string.greeting, userName))
```

## Method 1: Standard Android (Jetpack Compose)

Perfect for **Android-only** applications using the classic Android resources system.

### 📁 Project Structure

```
app/
├── src/
│   ├── main/
│   │   ├── java/
│   │   └── res/
│   │       ├── values/
│   │       │   └── strings.xml          # Default (English)
│   │       ├── values-es/
│   │       │   └── strings.xml          # Spanish
│   │       ├── values-fr/
│   │       │   └── strings.xml          # French
│   │       └── values-ja/
│   │           └── strings.xml          # Japanese
```

### 📝 Creating String Resources

**Default strings.xml** (`res/values/strings.xml`):
```xml
<resources>
    <string name="app_name">My Awesome App</string>
    <string name="hello_world">Hello World!</string>
    <string name="greeting">Hello, %1$s!</string>
    <string name="item_count">You have %1$d items</string>
    <string name="welcome_message">Welcome to %1$s, %2$s!</string>
</resources>
```

**Spanish translation** (`res/values-es/strings.xml`):
```xml
<resources>
    <string name="app_name">Mi Increíble App</string>
    <string name="hello_world">¡Hola Mundo!</string>
    <string name="greeting">¡Hola, %1$s!</string>
    <string name="item_count">Tienes %1$d elementos</string>
    <string name="welcome_message">Bienvenido a %1$s, %2$s!</string>
</resources>
```

### 💻 Using in Composable Functions

```kotlin
import androidx.compose.material3.Text
import androidx.compose.ui.res.stringResource
import com.yourapp.R

@Composable
fun WelcomeScreen(userName: String, appName: String) {
    Column {
        // Simple string
        Text(text = stringResource(R.string.hello_world))
        
        // Single parameter
        Text(text = stringResource(R.string.greeting, userName))
        
        // Multiple parameters
        Text(text = stringResource(R.string.welcome_message, appName, userName))
    }
}

@Composable
fun CartScreen(itemCount: Int) {
    Text(text = stringResource(R.string.item_count, itemCount))
}
```

## Method 2: Compose Multiplatform (KMP)

Best for **cross-platform** applications targeting Android, iOS, Desktop, and Web.

### 🔧 Setup

**1. Apply the plugin** in your `build.gradle.kts`:

```kotlin
plugins {
    kotlin("multiplatform")
    id("org.jetbrains.compose")
    id("org.jetbrains.kotlin.plugin.compose")  // For Compose 1.7+
}

kotlin {
    androidTarget()
    iosX64()
    iosArm64()
    iosSimulatorArm64()
    jvm("desktop")
    
    sourceSets {
        val commonMain by getting {
            dependencies {
                implementation(compose.runtime)
                implementation(compose.foundation)
                implementation(compose.material3)
                implementation(compose.ui)
            }
        }
    }
}

compose {
    resources {
        packageOfResClass = "com.yourapp.shared.resources"
        publicResClass = false  // Set to true if you want the Res class to be public
    }
}
```

**2. Create resources directory structure:**

```
commonMain/
├── composeResources/
│   ├── values/
│   │   └── strings.xml              # Default (English)
│   ├── values-es/
│   │   └── strings.xml              # Spanish
│   ├── values-fr/
│   │   └── strings.xml              # French
│   └── values-de/
│       └── strings.xml              # German
```

### 📝 Creating String Resources

**Default strings.xml** (`composeResources/values/strings.xml`):
```xml
<resources>
    <string name="app_name">My KMP App</string>
    <string name="welcome">Welcome, %s!</string>
    <string name="settings">Settings</string>
    <string name="profile">Profile</string>
</resources>
```

**Spanish translation** (`composeResources/values-es/strings.xml`):
```xml
<resources>
    <string name="app_name">Mi App KMP</string>
    <string name="welcome">¡Bienvenido, %s!</string>
    <string name="settings">Configuración</string>
    <string name="profile">Perfil</string>
</resources>
```

### 💻 Using in Composable Functions

**Important:** Rebuild your project after adding/editing strings to generate the `Res` class.

```kotlin
import androidx.compose.material3.Text
import org.jetbrains.compose.resources.stringResource
import com.yourapp.shared.resources.Res

@Composable
fun MyApp() {
    MaterialTheme {
        Column {
            // Simple string
            Text(text = stringResource(Res.string.app_name))
            
            // With parameter
            Text(text = stringResource(Res.string.welcome, "Alice"))
            
            // Navigation items
            Text(text = stringResource(Res.string.settings))
            Text(text = stringResource(Res.string.profile))
        }
    }
}
```

## Advanced Features

### 🎯 Plurals (Quantity-dependent strings)

**XML Definition:**
```xml
<resources>
    <plurals name="book_count">
        <item quantity="zero">No books</item>
        <item quantity="one">1 book</item>
        <item quantity="two">2 books</item>
        <item quantity="few">%d books</item>
        <item quantity="many">%d books</item>
        <item quantity="other">%d books</item>
    </plurals>
</resources>
```

**Usage:**
```kotlin
// Standard Android
Text(text = resources.getQuantityString(R.plurals.book_count, count, count))

// Compose Multiplatform
Text(text = stringResource(Res.plurals.book_count, count, count))
```

### 📊 String Arrays

**XML Definition:**
```xml
<resources>
    <string-array name="weekdays">
        <item>Monday</item>
        <item>Tuesday</item>
        <item>Wednesday</item>
        <item>Thursday</item>
        <item>Friday</item>
        <item>Saturday</item>
        <item>Sunday</item>
    </string-array>
</resources>
```

**Usage:**
```kotlin
// Standard Android
val weekdays = resources.getStringArray(R.array.weekdays)

// Compose Multiplatform
val weekdays = stringArrayResource(Res.array.weekdays).toList()
```

### 🎨 HTML Formatting

```xml
<string name="formatted_text">
    <![CDATA[
        <b>Bold text</b><br/>
        <i>Italic text</i><br/>
        <u>Underlined text</u><br/>
        <font color="#FF0000">Red text</font>
    ]]>
</string>
```

**Usage:**
```kotlin
// Standard Android
Text(
    text = buildAnnotatedString {
        append(stringResource(R.string.formatted_text))
    }
)
// Or use fromHtml() for HTML parsing
```

## Runtime Language Switching

### 🔄 Standard Android (using CompositionLocal)

```kotlin
import android.content.res.Configuration
import androidx.compose.runtime.Composable
import androidx.compose.runtime.CompositionLocalProvider
import androidx.compose.runtime.staticCompositionLocalOf
import java.util.Locale

data class LocaleState(
    val locale: Locale,
    val onLanguageChange: (Locale) -> Unit
)

val LocalLocale = staticCompositionLocalOf {
    LocaleState(Locale.getDefault(), {})
}

@Composable
fun LocaleAwareApp(content: @Composable () -> Unit) {
    val localeState = remember { mutableStateOf(Locale.getDefault()) }
    
    CompositionLocalProvider(
        LocalLocale provides LocaleState(
            locale = localeState.value,
            onLanguageChange = { newLocale ->
                localeState.value = newLocale
                // Update configuration
                val config = Configuration(LocalConfiguration.current)
                config.setLocale(newLocale)
                // Apply new config to activity context
            }
        )
    ) {
        content()
    }
}

// Usage in any composable
@Composable
fun SettingsScreen() {
    val localeState = LocalLocale.current
    
    Button(onClick = {
        localeState.onLanguageChange(Locale("es"))
    }) {
        Text(stringResource(R.string.change_language))
    }
}
```

### 🌐 Compose Multiplatform (using Lyricist)

Add dependency:
```kotlin
commonMain.dependencies {
    implementation("com.github.vanpra.compose-music-player:lyricist:0.2.0")
}
```

Implementation:
```kotlin
import com.vanpra.lyricist.Lyricist
import com.vanpra.lyricist.LocalLyricist
import com.vanpra.lyricist.material3.string

// Define your strings in Kotlin (no XML needed)
sealed class Strings {
    data class Welcome(val name: String): Strings()
    data class ItemCount(val count: Int): Strings()
}

@Composable
fun MyApp() {
    val lyricist = rememberLyricist(
        translations = mapOf(
            "en" to object : StringTranslations<Strings>() {
                override fun Strings.translate() = when(this) {
                    is Strings.Welcome -> "Welcome, ${name}!"
                    is Strings.ItemCount -> "You have $count items"
                }
            },
            "es" to object : StringTranslations<Strings>() {
                override fun Strings.translate() = when(this) {
                    is Strings.Welcome -> "¡Bienvenido, ${name}!"
                    is Strings.ItemCount -> "Tienes $count elementos"
                }
            }
        ),
        defaultLanguage = "en"
    )
    
    CompositionLocalProvider(LocalLyricist provides lyricist) {
        // app content
    }
}

@Composable
fun WelcomeScreen(name: String) {
    Text(text = string(Strings.Welcome(name)))
}
```

## Best Practices

### ✅ DO's

1. **Use meaningful resource names**
   ```xml
   <!-- Good -->
   <string name="login_button_text">Sign In</string>
   <string name="error_network_unavailable">No internet connection</string>
   
   <!-- Bad -->
   <string name="string1">Sign In</string>
   <string name="error">No internet</string>
   ```

2. **Group related strings**
   ```xml
   <!-- Login screen -->
   <string name="login_title">Welcome Back</string>
   <string name="login_username">Email or Username</string>
   <string name="login_password">Password</string>
   
   <!-- Profile screen -->
   <string name="profile_title">My Profile</string>
   <string name="profile_edit">Edit Profile</string>
   ```

3. **Use string resources for ALL user-facing text**
   - Even if you think you'll never translate
   - Makes future i18n much easier

4. **Provide translations for plurals and quantities**
   - Different languages have different plural rules

5. **Test with longer text**
   - German text is often 30% longer than English
   - Russian/Japanese can be much shorter or longer

### ❌ DON'Ts

1. **Don't concatenate strings**
   ```kotlin
   // Bad
   Text(text = stringResource(R.string.hello) + " " + userName)
   
   // Good
   Text(text = stringResource(R.string.hello_format, userName))
   ```

2. **Don't hardcode text in composables**
   ```kotlin
   // Bad
   Text("Hello World")
   
   // Good
   Text(stringResource(R.string.hello_world))
   ```

3. **Don't use images with text**
   - Use separate text overlays that can be translated

4. **Don't assume writing direction (LTR)**
   - Support RTL languages (Arabic, Hebrew, Persian)

## Troubleshooting

### 🔧 Common Issues and Solutions

| Issue | Solution |
|-------|----------|
| **Generated Res/R class not found** | Rebuild project (Build → Rebuild Project) |
| **Strings not updating** | Clean project and rebuild |
| **Parameter formatting not working** | Use `%1$s`, `%2$d` format specifiers |
| **Plurals showing wrong form** | Check all quantity types for your language |
| **Special characters not displaying** | Use CDATA sections or escape HTML entities |
| **Compose preview showing wrong language** | Set locale in preview: `@Preview(locale = "es")` |

### 🐛 Debug Tips

```kotlin
// Log current locale
val currentLocale = resources.configuration.locales[0]
Log.d("i18n", "Current locale: $currentLocale")

// Check available locales
val availableLocales = resources.assets.locales
Log.d("i18n", "Available: ${availableLocales.joinToString()}")

// Force specific locale (testing only)
val config = resources.configuration
config.setLocale(Locale("fr"))
resources.updateConfiguration(config, resources.displayMetrics)
```
